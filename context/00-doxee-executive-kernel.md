## Prioritization Framework (Extended RICE)

```
Value = ((Reach × Impact) + Cost_Reducing + Cofinancing) × (Confidence/100) / Effort
```

| Component | Scale | Key Detail |
|-----------|-------|-----------|
| Reach | 1-5 | % customers affected in year 1 (revenue-weighted) |
| Impact | 1-5 | 1=Tech Debt → 5=Game Changer |
| Confidence | 50/80/100% | Quantitative evidence required for High |
| Effort | 1-5 | Person-days (1=0-100, 5=400-500). >500 = must split |
| Cost Reducing | 1-5 | Annual savings (1=<€30K, 5=>€120K) |
| Cofinancing | 1-5 | Customer co-investment level (20-100%) |

## Constitution Principles (Derived)

1. Privacy by design — GDPR embedded in architecture
2. Accessibility-first — WCAG 2.1 AA by default
3. AI-first development — every feature considers AI augmentation from inception
4. API-first everything — every UI capability available via API
5. European sovereignty — data residency and regulatory compliance as foundational
6. Business user empowerment — IT governs, doesn't gatekeep
7. Sustainability by design — measure/optimize environmental impact
8. Cloud agnostic — support multiple providers and on-premise
9. Ecosystem integration — CRM/ERP as core features, not custom projects
10. Explainable and auditable — transparency in AI decisions
11. Journey context — every communication knows its place in customer journey
12. Balanced portfolio — competitive parity in core CCM, differentiation in engagement
