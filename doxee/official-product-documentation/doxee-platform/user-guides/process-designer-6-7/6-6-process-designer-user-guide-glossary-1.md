**Activity**

A predefined component which executes a predefined activity (e.g. rendering a document, transforming an XML file or reading from a database). An activity is included in a process via an activity element.

**Activity element**

A process element which executes a specific activity.

**Base directory**

See Process Engine base directory.

**Body**

See Process body.

**Connection**

See Process connection.

**Element**

See Process element.

**Header**

See Process header.

**Literal**

A constant string value stored directly in a process definition or config file.

**Module**

See Process Engine module.

**Parameterised literal**

A literal that may contain expressions enclosed in curly braces ({}).

**Physical process path**

The actual location of a process (or other Process Engine related resource).

**Process**

Depending on the context, a process definition or a process instance.

**Process body**

The part of a process definition that is made up of process components.

**Process component**

The building blocks of a process. Components are divided into connections and elements.

**Process connection**

A connection between two process elements.

**Process element**

A node in a process graph. Process elements include start and end elements; mapping, activity and script elements; groups, loops, decisions and waitstate and task elements.

**Process header**

The header section of a process definition, containing definitions which apply to the entire process.

**Process instance**

A single execution instance of a process definition.

**Process definition**

The structure of a process, as stored in an IPD file and typically created in Process Designer.

**Process Engine base directory**

The base directory of a Process Engine installation, where its configuration files are found. Files used by the Process Engine are usually located in the base directory or its sub directories, but configuration files may also specify additional external directories for certain resources.

**Process Engine module**

A Jar file (or a collection of Jar files) providing activities and/or XPath functions for the Process Engine. Additional modules can be installed to extend the functionality of a Process Engine installations.
